# 基于 Cordova Android 的源码定制改动记录

为方便后续追踪与新版本的演进迭代，文档记录了 `cordova-android` 官方源码仓库基础上所做的全部定制化修改、注入与拦截逻辑。目前的改动完全基于 `14.0.1`，并且已统一提交至 `custom/main` 或 `custom/14.0.1` 分支上维护。

## 一、新增或剥离的独立业务代码

为了保证对 Cordova 原始代码尽可能的低侵入性，所有并非纯框架运行本身需要，但为了方便宿主容器（如 `cordova-webcontainer` / `yumc` 等）感知和拦截状态业务，增加的代码均统一定义于以下目录：

**新增包路径**：`framework/src/org/apache/cordova/customer/`

1. **`constant.PluginMessageId.java`**
   - 定义了原版 Cordova 交互流程中未统一的一票魔法字符串，以及为了追踪状态新增的消息 ID（如 `pluginExecute`，`pluginResult`，`onNavigationAttempt`，`onProgressChanged` 等）。
2. **`data.PlugnExecute.java`**
   - 封装了前端（JavaScript）发起一次插件调用时所有的核心参数实体模型。
   - 包含的字段包括：触发的 `url`，插件配置名称 `pluginName`，插件的 Java 真实类名 `pluginClassName`，`callbackId`，调用的 `action` 以及 `rawArgs` 参数源。
3. **`data.PlugnExecResult.java`**
   - 封装了原生（Android）处理完 JavaScript 插件调用后回传的结果数据模型，便于提供给拦截层做埋点审计。
   - 包含的字段包括：`callbackId`，返回的状态码和错误消 `status`，以及携带的 `message`。
4. **`listener.PageScrollChangedListener.java`**
   - 定义了一个供外部注册的回调接口，以便于将内核底层真正的 WebView 滚动距离实时通知给 Cordova 的 Activity 或上层外壳容器。
5. **`CordovaApiInterceptor.java`**
   - 接口定义。向外界暴露了 `boolean shouldAllowApi(String url, String service, String action)` 钩子，专门用于白名单限制与拦截引擎上的不合规 API 访问。

## 二、Core Android 框架源码文件修改细节

### 1. 引擎执行与插件管道

**文件：`PluginManager.java`**
这是所有定制变更的核心：
- **新增属性**：
  引入了 `CordovaApiInterceptor` 和实时缓存最近活动页面地址的 `currentUrl` 变量。
- **改动 `postMessage`**：
  当框架捕捉到 `PluginMessageId.onPageStarted` 与 `PluginMessageId.onPageFinished` 时，更新当前缓存的 `currentUrl`（以解决多端切换带来的白名单匹配失误）。
- **改动 `exec`**：
  1. 在接收到真实的插件执行指令时，首先进行 `apiInterceptor` 匹配检查，若未通过 `shouldAllowApi` 的认证，就会触发 `ERROR` 中断报错 `API not allowed`。
  2. 通知订阅者即将执行任务本身。新增发送实体：`postMessage(PluginMessageId.pluginExecute, plugnExec);` （附带 `PlugnExecute` 等上下文环境属性）。

**文件：`CallbackContext.java`**
- **改动 `sendPluginResult`**：
  当原生方法返回任何状态给 Web 层时，在其完成返回之后新增回调通知 `postMessage(PluginMessageId.pluginResult, plugnExecFinish);` 。由于这是唯一能监控插件处理完成的地方，增加这一步就能与上述 `PluginManager.exec()` 呼应形成完整的插件调用链路监控和闭环。

---

### 2. WebView 解析与代理生命周期

**文件：`engine/SystemWebView.java`**
- **增加视图滚动回调**：
  重写了 Android View 原生的 `onScrollChanged(int l, int t, int oldl, int oldt)`。如果上层注入了 `PageScrollChangedListener`，将在这里向它发送所有的滚动坐标变化，以供宿主判断顶部栏伸缩或刷新动作。

**文件：`engine/SystemWebChromeClient.java`**
- **扩充宿主监控维度（标题和进度）**：
  在 `14.0.1` 官方基座上没有暴露对于 ChromeClient 的默认通知。此处重写并新增了如下实现：
  1. `onProgressChanged` 收到变更后通过 `postMessage(PluginMessageId.onProgressChanged, newProgress)` 下发进度。
  2. `onReceivedTitle` 收到变更后通过 `postMessage(PluginMessageId.onReceivedTitle, title)` 下发最新的网页标题内容（方便上层导航条更新 `title`）。

**文件：`CordovaWebViewImpl.java`**
- **消息发送的重构与补充通知**：
  1. 在 `onNavigationAttempt` 内拦截、重写、和请求判断链中全部增强增加了 `postMessage`：发送如 `PluginMessageId.onNavigationAttempt`、`PluginMessageId.shouldAllowNavigation`、`shouldOpenExternalUrl` 和 `onOverrideUrlLoading` 的探测。
  2. 将其余原先硬编码（`"onPageStarted"`，`"spinner"`等）全都替换为了 `PluginMessageId.*` 常量引用。

---

### 3. Cordova 活动基类适配

**文件：`CoreAndroid.java`，`CordovaActivity.java`，`CordovaInterfaceImpl.java`**
- **事件常量重构兼容**：
  统统统一将原有的、通过 `String` 直接拼写的 `"exit"`，`"onReceivedError"` 取代为 `customer.constant.PluginMessageId.*`，避免代码冗余和错拼失误。

## 三、Gradle 编译构建脚本变更

为了能脱离源码树的限制支持将 `cordova-android` 独立打包分发至远端：

**文件：`framework/build.gradle.kts`**
- 将原本过时的 Groovy `build.gradle` 直接迁移到 `kts` 实现。
- 引入 `com.vanniktech.maven.publish` 插件处理产物的签名验证及 POM 信息组装并对外输出，让上层外壳不需要复制 `framework` 文件夹也能通过 Maven 坐标拉取该版本的包。

**文件：`framework/gradle.properties`**
- 注入远端发行必须的参数设置以及坐标声明：包括 `GROUP`（例如 `com.xeonyu`）、`POM_ARTIFACT_ID` 和 `VERSION_NAME` 等等。

**文件：`test/androidx/app/build.gradle` & 兄弟文件**
- 为跑通验证功能修改了测试用脚手架内部的配置文件，解决了 Kotlin Standardlib 等因为 AGP 升级产生的依赖冲突兼容问题。
